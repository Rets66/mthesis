\subsubsection{評価要素 : RTT(Round Trip Time)}
従来のDNSでは，ラベルごとにゾーンが移譲されている場合，レコード情報を保持するノードまでのホップ数はラベル数nに対してO(n)になる．
対して，REFRESでは，識別子から一意にレコード情報を保持するマネージャノードを特定できるので，O(2)のホップ数となる．
このため，従来よりも速度の向上が期待される．
